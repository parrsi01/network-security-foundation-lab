# Lesson Execution Companion

Author: Simon Parris  
Date: 2026-02-26

Use this companion while running labs so the output becomes portfolio-quality evidence, not just task completion.

## Per-Lab Checklist

1. State the objective in one sentence.
2. Identify the expected observable(s) before running commands.
3. Run the documented steps exactly once before modifying anything.
4. Record key evidence (alerts, packet traces, protocol fields, rule hits).
5. Run the intentional failure/noise scenario if included.
6. Use the troubleshooting tree before reading the full solution.
7. Validate with the lab script.
8. Write a short analysis note: signal, noise, risk, next action.

## Analyst Prompts

- What evidence supports this as malicious vs benign?
- What additional telemetry would reduce uncertainty?
- What assumptions am I making about network topology?
